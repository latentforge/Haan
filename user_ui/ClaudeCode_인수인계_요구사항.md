# AI 카드상담 서비스 — Claude Code 인수인계 문서

## 0. 한 줄 요약
카드사 고객이 **전화 통화**로 AI와 상담하고, 그 통화 내용이 **상담회사(상담사)용 UI**에 실시간 전사되며, 화면 오른쪽에 **AI가 판단하고 도구를 실행하는 흐름**이 라이브로 표시되는 서비스. 지금 단계는 **껍데기(목업) 우선 구현** — 실제 음성엔진·자체 LLM·카드사 시스템은 나중에 연결한다.

---

## 1. 구성 (앱이 2개다)

**A. 고객용 모바일 앱** (iOS + Android)
- 첨부된 `user.html` = 이 화면. AI와 음성 통화하는 오브(구슬) 화면.
- 진짜 전화 통화 기반.

**B. 상담회사용 UI** (상담사가 보는 화면)
- 왼쪽: 고객 발화가 **실시간 전사**되어 표시.
- 오른쪽: **"AI 판단 및 실행 흐름"** 패널 (별도 HTML 첨부 예정).
- ⚠️ 확인 필요: 이 화면은 상담사가 PC/태블릿에서 보는 **웹 화면**으로 가정함(모바일 앱 아님). 아니면 알려주세요.

---

## 2. 핵심 동작 흐름 (데모 시나리오)
1. 고객이 앱에서 통화 시작 → 말함.
2. 음성이 글자로 전사됨 → **상담사 UI 왼쪽**에 표시.
3. AI가 발화를 분석 → **본인확인**을 먼저 수행.
4. **본인확인이 완료되면**, AI가 카드사 개인정보에 접근 가능해짐(= 이 편리함을 보여주는 게 데모의 핵심 포인트).
5. AI가 판단 과정(감지→단서→본인확인→도구 호출)을 **오른쪽 패널에 라이브로** 표시.
6. 필요 시 **카드정지 / 분실신고 접수** 도구 실행.

---

## 3. 지금 만들 것 vs 나중에 붙일 것 (매우 중요)

| 항목 | 지금 | 방식 |
|---|---|---|
| 음성 인식(STT)·합성(TTS) | ❌ 만들지 않음 | **우리(사용자)가 자체 구현 중.** Claude Code는 "전사 텍스트가 들어온다"고 가정하고, **입력 받는 인터페이스(어댑터)만** 만들 것 |
| AI 모델(대화·판단) | ❌ 아직 없음 | **자체 모델 예정.** 지금은 **Mock LLM 어댑터**로 대체 — 나중에 모델을 끼우면 바로 교체되도록 인터페이스 분리 |
| 카드사 시스템 연동 | ❌ 없음 | **가짜(Mock) 데이터**로 흉내 |
| 본인확인/카드정지/분실신고 | ✅ 껍데기 구현 | Mock API 엔드포인트 + 상태 전환(대기→실행중→완료)까지 실제처럼 동작 |
| 두 화면 UI | ✅ 구현 | 첨부 HTML 기반, 실시간 연동 |

> 설계 원칙: **어댑터 패턴**으로 "가짜 부품"을 끼워두고, 실제 음성엔진·모델·카드사 API가 오면 **그 부품만 교체**하면 되게 만들 것. (자동차에 임시 엔진을 달아두고, 진짜 엔진 오면 갈아끼우는 구조)

## 4. 도구(Tool) 명세 — Mock 계약
아래 3개를 Mock API로 구현. 요청/응답은 예시이며 Claude Code가 실제처럼 상태를 흐르게 할 것.

**verify_identity (본인확인)**
```
요청:  { name, birth, phone_last4 }
응답:  { verified: true, customer_id: "MOCK-1001", unlocked: true }
※ verified=true가 되면 이후 개인정보 조회 도구가 열림
```
**suspend_card (카드정지)**
```
요청:  { customer_id, card_id }
응답:  { status: "suspended", card_id, at: "<timestamp>" }
```
**report_lost (분실신고 접수)**
```
요청:  { customer_id, card_id, memo }
응답:  { report_id: "MOCK-RPT-2001", status: "received" }
```
가짜 고객/카드 데이터(예: 김민서, 끝자리 4821, 카드 2개)를 seed 데이터로 둘 것.

---

## 5. 두 화면 실시간 연결
- 고객 앱 ↔ 상담사 UI는 **WebSocket**으로 이벤트를 주고받음.
- 이벤트 예: `transcript`(전사 한 줄), `ai_step`(판단 노드 1개), `tool_call`(도구 상태 변화).
- 오른쪽 패널은 이 이벤트를 받아 **노드가 하나씩 스트리밍**되도록(앞서 정의한 동적 UI 규칙 그대로).

---

## 6. 인증·보안 (데모 컨셉)
- 원래는 로그인이 필요한 서비스지만, **데모의 메시지**는 "AI가 본인확인만 해주면 카드사 개인정보에 바로 접근되는 편리함".
- 그래서: 별도 로그인 화면 없이, **본인확인 도구 성공 = 개인정보 잠금 해제**로 표현.
- ⚠️ 실서비스 전환 시엔 실제 인증·암호화·개인정보 동의가 필요하다는 주석을 코드에 남길 것. (데모용임을 명확히)

## 7. 추천 기술 스택 (10번 답변 포함)

**고객 앱 (iOS+Android)** — 이미 정교한 HTML/CSS/WebGL 오브가 있으니, 이걸 버리지 말고 **네이티브 껍데기로 감싸는** 방식 추천:
- **Capacitor(또는 Ionic)** 로 `user.html`을 그대로 iOS/Android 앱으로 패키징.
- 전화 통화·마이크는 네이티브 플러그인으로 연결(음성엔진은 우리가 붙일 자리만 마련).
- 장점: 지금 디자인을 픽셀 그대로 유지하면서 두 OS 동시 지원.

**상담사 UI (웹)**
- **React + Vite + Tailwind** + WebSocket. 첨부될 AI 판단 흐름 HTML을 컴포넌트화.

**Mock 백엔드**
- **Node.js + Express + WebSocket(ws)**. 위 도구 3종 + 가짜 데이터 + Mock LLM 어댑터.

**배포 (추천)**
- 상담사 UI: **Vercel** (웹이라 가장 간단).
- Mock 백엔드: **Railway** 또는 **Render** (WebSocket 지원, 무료/저가 티어로 데모 충분).
- 고객 앱: 데모 단계엔 **TestFlight(iOS) / Play 내부테스트(Android)** 로 배포. 정식 스토어 등록은 나중.
