## 0. 읽기 전에 — 현재 구현과 최종 구현의 구분

**Haan**이라는 이름과 페르소나(恨/限/韓, "Hearing Always, Acting Naturally")는 프로젝트 전체를 관통하는 **정체성**이며 두 단계 모두에 걸쳐 유지된다. 다만 그 정체성을 실현하는 **기술적 실체는 두 단계로 나뉜다**. 본 문서에서 기능을 서술할 때는 항상 아래 구분을 따른다.

| 구분 | 현재 구현 (이 레포 = Haan) | 최종 구현 (최종 연구 목표) |
|---|---|---|
| **아키텍처 계열** | Moshi 기반 Full-duplex (Mimi 코덱 + Temporal/Depth Transformer) | **TITANS** 계열 (MAE, Memory as Embedding) |
| **백본** | Qwen3-8B로 교체 | (최종 설계 시 확정) |
| **핵심 과제** | 한국어 멀티턴 데이터 없이 turn-taking **전이**로 한국어 Full-duplex 창발 | 무제한 문맥 기억(3분의 벽 극복) + 실시간 Tool-Calling |
| **문맥/기억** | Moshi 원 설계의 컨텍스트 한계를 그대로 상속(약 수 분) | O(1) 메모리로 장시간 세션 기억 내재화 |
| **문서 근거** | `PROJECT.md`, `ARCHITECTURE.md`, `TRAINING_CURRICULUM.md`, `DATA_STRATEGY.md`, `RISKS_AND_DIAGNOSTICS.md` | 본 문서 §3.2 및 별도 최종 설계 문서(예정) |

> **요약**: 이 레포(`Haan`)에서 실제로 구현·학습하는 것은 **Moshi+Qwen3 기반 한국어 Full-duplex 전이 모델**이다. TITANS/MAE·무제한 기억·Tool-Calling은 **최종 구현이 지향하는 목표**로, 아직 현재 레포의 구현 대상이 아니다. 아래 각 섹션은 이 둘을 명시적으로 갈라서 기술한다.

## 1. 이름

**Haan**
(로마자 표기, 발음: /haːn/ — "한"을 늘여 발음)

- 한국어 "**한**"의 장모음 표기. "하나(Hana)"와 시각적·발음적으로 명확히 구분됨
- 외국인도 무리 없이 발음 가능한 1음절 구조 (H + long-a + n)
- 국내에서 "한 모델"이라는 발화가 "하나의 모델"과 혼동되는 문제를 피하기 위해 장모음화된 표기를 채택

## 2. 한(漢字) 삼중 의미 구조

이름 "Haan"은 하나의 소리에 세 가지 한자·개념을 동시에 담는다.

### 恨 (한) — 풀어야 할 감정
> 반복되는 오해, 끊기는 대화, 쌓여가는 억울함

2024년 하반기 은행권 AI 상담 도입 이후 실제로 벌어진 일들 — 콜봇 이용 고객의 52%가 상담을 끝까지 마치지 못하고 중단했고, 그중 53%는 불만족스러워했다. 상담사는 짧은 기억력의 AI가 초래한 고객의 분노를 대신 받아내며 "숨소리를 내는 것도 조심스럽다"고 말한다.

**Haan은 이 恨을 풀어주는 기술이다.** 고객이 같은 설명을 3~4번 반복하지 않아도 되는 대화, 상담사가 AI의 실수를 대신 사과하지 않아도 되는 구조.

### 限 (한) — 넘어서야 할 경계
> "3분의 벽"이라는 구조적 한계

GPT-4o, Gemini, Qwen3-Omni 등 현재 상용 AI는 약 3분 내외의 대화만 기억한다. 복잡한 금융 상담은 평균 1시간이 걸린다. 기존 Transformer의 KV 캐시는 대화 길이에 비례해 선형 증가(O(n²))하다가 초기 맥락을 지워버린다.

**Haan이 최종적으로 이 限을 없앤다.** — 단, 이는 **최종 구현의 목표**다. **TITANS MAE(Memory as Embedding)** 아키텍처로 과거 정보를 고정 크기의 임베딩 공간에 고밀도로 내재화해, 몇 시간 혹은 며칠에 걸친 세션에서도 거리에 관계없이 동일한 강도로 맥락을 기억하는 것을 지향한다.

> **현재 구현(이 레포) 상태**: 현재 Haan은 Moshi 기반 Full-duplex 구조를 상속하므로 문맥 기억은 아직 Moshi 원 설계 수준(약 수 분)에 머무른다. "3분의 벽" 극복(무제한 기억)은 TITANS 계열로 전환하는 최종 구현에서 달성 대상이며, 현 단계의 컨트리뷰션은 §0 표의 "현재 구현" 열(한국어 turn-taking 전이)에 집중되어 있다.

### 韓 (한) — 지켜야 할 정체성
> 미·중 주도 생태계에 대한 독자적 대안

TITANS는 아직 학술적 표준이 정립되지 않은 아키텍처다. Haan은 한국어 맥락에 최적화된 독자적 전이중(Full-Duplex) 상담 엔진으로서, 글로벌 기술 종속에서 벗어나 국내 상담·콜센터 산업의 기술 자립도를 높이는 것을 목표로 한다.

## 3. 무엇을 하는 모델인가

Haan의 지향점은 **한국어에 최적화된 Full-Duplex 음성 상담 AI**이며, 이 지향점을 **현재 구현**과 **최종 구현** 두 단계로 나누어 실현한다. 아래 §3.1과 §3.2는 §0의 구분에 각각 대응한다.

### 3.1 현재 구현 (이 레포 = Haan) — Moshi 기반 한국어 Full-Duplex 전이

현재 레포에서 실제로 구현·학습하는 실체. Kyutai Moshi 아키텍처(Mimi 코덱 + Temporal/Depth Transformer)를 백본만 Qwen3-8B로 교체해 재사용하고, **한국어 멀티턴 대화 데이터 없이** turn-taking 능력을 전이시키는 것이 핵심 과제다. (근거: `PROJECT.md`, `ARCHITECTURE.md`, `TRAINING_CURRICULUM.md`, `DATA_STRATEGY.md`)

| 축 | 내용 |
|---|---|
| **듣기와 말하기의 동시성** | 반이중(Half-Duplex)의 한계 — 사용자 발화 완료 후에야 추론 시작 — 를 탈피. Moshi의 self/user 이중 스트림 병렬 예측으로 듣는 동시에 말한다 |
| **한국어 turn-taking 창발** | 영어 멀티턴 KD(원본 Moshi teacher) + 한국어 싱글턴 TTS의 Joint Loss로, 한국어 멀티턴 데이터 없이 침묵·끼어들기·맞장구 타이밍을 전이 |
| **언어 학습 비용 제거** | 이미 한국어를 아는 Qwen3-8B 백본에 오디오 모달리티만 접합 — "언어를 가르치는" 비용을 백본 교체로 생략 |
| **문맥 범위(현 상태)** | Moshi 원 설계를 상속하므로 아직 수 분 수준. 장시간 기억은 최종 구현(§3.2)의 과제 |

### 3.2 최종 구현 (최종 연구 목표) — TITANS 기반 Any2Any 시스템

향후 최종 연구에서 전환할 목표 아키텍처. **TITANS 메모리 아키텍처를 기반으로 한 Any2Any Full-Duplex 멀티모달 시스템**을 지향한다. (아직 이 레포의 구현 대상이 아님)

| 축 | 내용 |
|---|---|
| **듣기와 말하기의 동시성** | 1D Convolution 기반 채널 분리로 듣는 동시에 말하고, 말하면서 동시에 작업을 실행 |
| **무제한 문맥 처리** | MAE(Memory as Embedding) 구조로 O(1) attention 연산량 유지. 기존 KV 캐시 대비 메모리 효율 3배 이상 향상. Test-time Training으로 추론 중에도 실시간 학습 |
| **끼어들기·의도 변경 대응** | 'Surprise Metric'과 'Forget Gate'로 중요 정보를 식별해 즉시 메모리에 반영. 모순 없는 자연스러운 대응 |
| **실시간 행동 생성 (Tool-Calling)** | 음성 응답 생성과 동시에 텍스트 기반 도구 호출을 수행하는 독립 채널 운영. "계좌 잔액을 확인해줘"라는 요청에 답변과 동시에 API를 호출해 응답 종료 시점에 정확한 정보 제공 |
| **초저지연** | End-to-End 100ms 이하의 응답 속도로 인간의 자연스러운 반응 속도 확보 |

## 4. 왜 지금 필요한가 — 기술 부채의 청산

> "AI 상담원 도입 후 근무 여건이 악화됐다" — 66%
> "AI 응답 오류로 인한 고객 민원 스트레스가 증가했다" — 53%

인건비 절감을 위해 도입된 불완전한 턴제 AI가 만들어낸 고객의 분노는, 결국 인간 상담사에게 감정 노동으로 전가되어 왔다. Haan은 이 악순환을 기술로 끊어내는 것을 목표로 한다.

- **상담원의 심리적 안전망**: '분노한 고객(Angry Customer)' 비율을 낮춰 상담사가 존중받으며 전문 업무에 집중할 수 있는 환경 조성
- **실질적 업무 완결**: 답변만 반복하는 수준을 넘어, AI가 실시간 Tool-Calling으로 업무를 직접 완결 (최종 구현 §3.2 목표)
- **디지털 약자 배려**: 턴제 방식의 기계적 대화에 적응하기 어려운 노약자·장애인이 인간과 대화하듯 편안하게 소통할 수 있는 인터페이스 제공

이는 단순한 효율성 증대가 아니라, 서비스 품질 저하의 책임을 상담원에게 떠넘기던 기술적 악순환을 끊어내는 **"따뜻한 기술"**의 실증 사례를 지향한다.

## 5. 서사적 연결 — theRiverLethe와의 관계

프로젝트의 기존 오픈소스 라이브러리명은 **theRiverLethe** (망각의 강, 그리스 신화 속 저승의 강).

> **Lethe(망각)의 강을 건너, Haan(恨/限을 넘어선 기억)에 도달한다.**

Lethe가 "모든 것을 잊게 하는 강"이라면, Haan은 그 반대편에서 "모든 것을 기억하고, 그 기억으로 쌓인 恨을 풀어내는" 모델이다. Amnesty(사면/기억상실이라는 이중적 프로젝트명)라는 이름 아래에서, Haan은 망각이 아닌 **온전한 기억을 통한 화해**를 상징한다.

## 6. 백로님 (Acronym)

### Haan — Hearing Always, Acting Naturally

| 구성 | 대응 기능 |
|---|---|
| **H**earing **A**lways | 능동적 청취. 타사 모델(RaonSpeech 등)이 연산량을 줄이기 위해 유저 입력을 선별적으로 무시하는 것과 달리, Haan은 유저의 말을 항상 청취하며 의도 변화에 즉각 반응한다 |
| **A**cting **N**aturally | 자연스러운 행동 생성. 턴제 방식의 부자연스러운 지연을 탈피해, Full-Duplex 구조로 듣기·말하기·행동이 자연스럽게 동시에 일어난다 (행동=Tool-Calling은 최종 구현 §3.2 목표, 현재 구현은 듣기·말하기 동시성까지) |

슬로건: *"Haan — Hearing Always, Acting Naturally."*
