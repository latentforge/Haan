# Haan 후속 연구 로드맵: Role 구분 메커니즘 재설계

이 문서는 Moshi의 self/user 오디오 임베딩 분리 구조를 재설계하는 두 개의
독립적이면서 계열이 이어지는 논문 계획을 정리한다. 두 논문은 서로를
전제하지 않고 각자 완결된 기여를 갖되, 같은 아이디어 계열(role 구분을
임베딩 분리가 아닌 별도 신호로 옮긴다)에서 파생된 후속 관계를 가진다.

---

## 배경: 문제의 정확한 위치

Moshi(및 이를 초기화로 쓰는 PersonaPlex)의 RQ-Transformer 구조에서, 한
타임스텝의 Temporal Transformer 입력은 다음과 같이 구성된다.

```
input_t = TextEmb(text_t) + Σ_k SelfAudioEmb_k(A_self[t,k]) + Σ_k UserAudioEmb_k(A_user[t,k])
```

여기서 `SelfAudioEmb_k`와 `UserAudioEmb_k`는 codebook k(semantic k=1,
acoustic k=2..8)마다 role별로 **완전히 분리된 학습 파라미터**다. 같은 RVQ
코드 인덱스라도 self 스트림에서 나왔는지 user 스트림에서 나왔는지에 따라
독립적으로 학습되는 별개 벡터를 참조한다.

**문제**: 한국어 학습 데이터가 self(모델 발화, TTS 등)에 집중되면
`SelfAudioEmb`는 한국어에 맞게 조정되지만 `UserAudioEmb`는 학습 노출이
적어 원래(영어 편향) 상태에 가깝게 남는다. 즉 self가 배운 한국어 인식/
표현 능력이 user 인식 경로로 전이되지 않는다.

**레이어 구분 (중요)**: 언어를 실제로 "배우는" 주체는 embedding table이
아니라 RQ-Transformer(Temporal + Depth)의 attention/FFN 가중치다.
Embedding table은 코드 인덱스에 대한 룩업 벡터를 제공할 뿐이다. 임베딩
공유가 기여하는 것은 "Transformer가 self 학습으로 획득한 처리 능력을,
user 입력 벡터에도 적용 가능하게 만드는 통로"이지, 임베딩 자체가 언어를
학습하는 것이 아니다.

**Temporal vs Depth 구분 (중요)**: user 오디오 이해는 전적으로 Temporal
Transformer의 역할이다(과거 프레임 임베딩 합을 문맥으로 압축). Depth
Transformer는 self 생성에만 관여하며 user 코드를 직접 입력받지 않는다.
따라서:
- **Semantic embedding 공유**: Temporal의 user 인식 능력에 직접 기여 →
  이득이 크고 명확함
- **Acoustic embedding 공유**: "생성 자연스러움이 user 쪽에도 전이된다"는
  기대는 성립하지 않음(Depth가 애초에 user 코드를 안 봄). 다만 Temporal이
  user의 paralinguistic 뉘앙스(감정, 강조 등)를 이해하는 데는 제한적으로
  기여할 수 있음 — semantic 대비 약한 부차 효과.
- 최종적으로는 semantic/acoustic 모두 공유하는 쪽으로 결정. acoustic
  공유의 기대 효과가 제한적이라는 점은 논문 1의 분석/ablation에서 정직하게
  다룬다.

**해법의 공통 전제**: 오디오 임베딩 테이블은 self/user 간 완전 공유하고,
role 구분은 임베딩이 아닌 별도의 신호(위치 인코딩 계열)로 옮긴다. 이
지점부터 두 논문이 갈라진다.

## 논문 1: Role Token — 한국어 전이를 위한 임베딩 통합

### 컨트리뷰션
Moshi의 self/user 오디오 임베딩 분리가 저자원 언어 전이를 저해함을
규명하고, 임베딩을 완전 공유하되 role 구분을 이진 role token(학습 가능한
additive 벡터, self/user 각 1개)으로 대체하는 구조를 제안. 단일 화자
발화(멀티턴이 아닌) 데이터만으로도 이해(user 인식)와 생성(self 발화)
양쪽에 한국어 능력이 전이됨을 실증.

### 설계
- 오디오 임베딩 테이블: self/user 공유, semantic + acoustic 전체
- Role 구분: `RoleEmb[role_id]` (role_id ∈ {0=self, 1=user}) 를 오디오
  임베딩에 additive
- 시간축 위치 인코딩(기존 RoPE)은 그대로 유지 — 이 논문에서 건드리지 않음

### 검토했으나 기각/보류한 대안
- **Dual PE를 시간축 개념으로 오인한 초기 논의**: role 구분을 시간축
  RoPE 자체를 role별로 다르게 쓰는 방식으로 검토했으나, 이는 self/user
  간 동기화(같은 timestep이라는 관계)를 깨뜨릴 위험이 있어 기각. Role은
  애초에 시간과 무관한 태그이므로 시간축 RoPE와는 별도 메커니즘이어야
  한다는 점을 확인.
- **Dual PE(학습형 role embedding) vs 3축 M-RoPE(role을 회전 축으로 고정)**:
  둘 다 "임베딩 공유 유지 + role만 구분"이라는 목적에는 대체로 동등하다고
  결론. Role embedding은 구현이 간단하고 기존 RoPE 체계(Qwen3 백본)를
  건드리지 않는다는 이점이 있어 논문 1의 기본안으로 채택. 정교한 비교는
  ablation으로 다룸.

### 검증해야 할 선행 리스크
- Mimi 코드북 자체가 한국어 음향(음소, 억양 패턴)을 담을 용량이 있는지는
  Transformer 레이어 이전 단계의 문제로, 임베딩 공유 여부와 무관하게
  별도로 sanity check 필요 (한국어 음성 왕복 인코딩/디코딩 재구성 품질
  측정 등).

### Ablation 설계 후보
1. 원 Moshi 구조(self/user 완전 분리) — baseline
2. 임베딩 공유 + Role Token (제안)
3. (참고) 임베딩 공유 + Dual PE 대안과의 비교
4. Semantic만 공유 vs semantic+acoustic 모두 공유

## 논문 2: Role-axis RoPE — 다중 화자로의 구조적 외삽

### 컨트리뷰션
논문 1의 "role 구분을 임베딩이 아닌 별도 신호로 분리한다"는 아이디어를
이어받아, 그 신호를 학습형 additive(role token)가 아닌 **결정론적 회전
축**으로 구현하면, 이진 self/user 구분을 넘어 임의 개수의 동시 화자를
자연스럽게 표현할 수 있음을 제안 및 실증. Moshi의 근본적 제약(고정
2-스트림)을 해소하는 아키텍처 확장.

이 논문은 한국어 전이 문제와 독립적으로 성립하며, 원본 Moshi 구조에서
바로 출발해도 무방하다. 논문 1과는 "같은 계열의 아이디어에서 나온 후속
연구"라는 관계이지, 논문 1의 결과에 종속되는 관계가 아니다.

### 설계
- Temporal Transformer의 RoPE를 두 축으로 분해:
  - **Temporal 축**: 기존과 동일, self/user(및 추가되는 모든 화자
    슬롯) 공통, 동일 프레임 인덱스 t 사용 — 시간 동기화 보존
  - **Role 축**: 화자 슬롯 인덱스에 따른 고정 회전 오프셋
    - 슬롯 0 (self): 오프셋 0
    - 슬롯 1 (user1): 오프셋 Δ
    - 슬롯 2 (user2): 오프셋 2Δ
    - ... 슬롯 n: 오프셋 nΔ
- 학습 시 모델은 항상 슬롯 0으로만 응답하도록 학습(self는 항상 0번
  고정). 그 외 슬롯은 전부 "비-self(상대방)"으로 취급.
- Qwen2.5-Omni의 TMRoPE(시간축 공유 + 보조축 분리로 오디오/비디오 동기화)가
  선례로서 이 설계의 타당성을 뒷받침.

### 핵심 주장: 왜 데이터 없이(1:1 데이터만으로) 다중 화자로 외삽되는가
- 슬롯 구분이 학습되는 표상이 아니라 기하학적으로 고정된 회전이므로,
  모델이 실제로 학습해야 하는 것은 "오프셋이 0인가 아닌가"라는 **단순
  이진 판단 하나**뿐이다. 오프셋이 Δ인지 2Δ인지 3Δ인지를 구분하는 능력은
  모델이 self 여부를 판단하는 데 필요조차 없다.
- RoPE의 attention score는 상대 회전각에 대해 매끄럽게(smooth) 변하는
  함수이므로, "0 근방이면 self, 0에서 멀수록 non-self"라는 판단은 특정
  회전각 값에 과적합되지 않고 자연스럽게 일반화된다.
- 이는 RoPE가 학습 시퀀스 길이를 넘어서는 거리에도 일반화되는(길이
  외삽) 것과 동일한 성질을 화자 슬롯 축에도 적용하는 것 — "다중 화자를
  시간축과 같은 종류의 축으로 다루면, 시간축 일반화 연구 성과를 그대로
  재사용할 수 있다"는 것이 이 논문의 핵심 통찰.
- 결과적으로 **1:1 대화 데이터만으로 학습해도**, 추론 시 슬롯을 2, 3, ...
  으로 늘려 멀티파티 대화에 투입 시 추가 학습 없이(혹은 최소 파인튜닝만
  으로) 화자 구분이 가능할 것으로 예상.
- Additive 학습형 role embedding(논문 1의 Role Token) 대비 이 성질이
  더 강하게 보장되는 이유: 학습형 벡터는 학습 중 실제로 본 슬롯 번호에
  최적화되므로 안 본 슬롯 번호(예: 슬롯 2)에 대한 일반화가 이론적으로
  덜 보장됨. 회전 기반은 슬롯 번호가 연속적인 파라미터(오프셋의 배수)이므로
  구조적으로 외삽에 더 유리.

### 남은 엔지니어링 과제 (설계 리스크 아님)
- 실시간 대화에서 새로 끼어드는 화자를 어떤 슬롯에 동적으로 배정할지
  (diarization 연동, 또는 발화 순서 기반 동적 할당)
- 슬롯 최대 개수 N을 얼마로 둘지 (오프셋 Δ 크기와 회전 주기 충돌 방지)
- 텍스트 토큰(기존 1D RoPE)과 오디오 토큰(role 축 추가) 간 회전 체계
  일관성 검증 — 백본이 순수 RoPE(Qwen3)로 사전학습되어 있으므로 사후
  개조에 해당, 별도 검증 필요

### 검증 계획 초안
- 논문 1과 같은 계열의 체크포인트(혹은 원 Moshi)를 role-axis RoPE로
  1:1 데이터만으로 학습
- 추론 시 슬롯 2 이상(멀티파티, 실제 3인 이상 동시 대화 코퍼스 또는
  합성 데이터)에 대해 zero-shot / few-shot 화자 구분 및 응답 적절성 평가
- Baseline: 논문 1의 Role Token 방식(이진 고정)이 슬롯 확장 시 어떻게
  실패하는지와 대비

## 두 논문의 관계 요약

| | 논문 1 (Role Token) | 논문 2 (Role-axis RoPE) |
|---|---|---|
| 핵심 문제 | 저자원 언어(한국어) 전이 | 화자 수(dyad → multi-party) 확장 |
| Role 구분 신호 | 학습형 additive 벡터 (이진) | 결정론적 회전 오프셋 (N-way) |
| 출발점 | PersonaPlex/Haan 초기화, 한국어 데이터 | 원 Moshi에서 독립적으로 시작 가능 |
| 컨트리뷰션 성격 | Data efficiency | Architectural generality |
| 상호 의존성 | 없음 — 각자 완결된 기여, 같은 아이디어 계열의 후속 연구 |
