# 상담사 AI UI (aiui.html) — Claude Code 구현 프롬프트

> 대상 파일: `aiui.html` (상담회사/상담사가 보는 화면).
> 이 문서 + `aiui.html`을 같이 넘기고 **"먼저 이 문서 읽고 aiui.html을 기준으로 구현해줘"** 라고 지시.

## 0. 이 화면이 뭔가
카드 상담 통화가 진행되는 동안 상담사가 태블릿에서 보는 실시간 작업 화면. 3개 패널로 구성:
1. **현재 통화** — 고객·AI·상담 발화가 실시간 전사되어 쌓임.
2. **상담 작업대** — 고객 정보, 본인확인 진행(1/3 체크리스트), 카드 후보.
3. **AI 판단 및 실행 흐름** — AI가 실시간으로 사고→도구 호출하는 과정을 라이브로 표시.

## 1. 이미 되어 있는 것 (그대로 유지 · 디자인 손대지 말 것)
`aiui.html`에 아래가 **이미 잘 구현**돼 있으니 새로 만들지 말고 재사용:
- 3패널 레이아웃, 색상(라벤더·민트 파스텔, oklch 토큰), SUIT 폰트, 반응형(1100px 이하 세로 스택).
- 오른쪽 패널의 동적 재생 로직: `aiThinkingEvents` 배열 → 타이핑(`typeText`), 카운트업(`countUp`, 0→87%), 분기선 그리기(`branchDraw`), 도구 상태 전환(`setToolStatus`: 실행 준비→실행 중→완료), 본인확인 자동 체크(`completePhone`), 워크플로 단계 강조 이동, 자동 스크롤, 다시 재생.

> 즉 이 화면의 "동적 사고 흐름"은 **이미 완성된 데모**다. Claude Code가 할 일은 **가짜 데이터를 실제 이벤트로 교체**하는 것이지, UI를 다시 그리는 게 아니다.

## 2. 지금 목업(가짜) → 실제 연동으로 바꿀 것

| 부분 | 지금(목업) | 바꿀 것 |
|---|---|---|
| 통화 전사(현재 통화) | HTML에 하드코딩된 발화 4줄 + "말하는 중" | STT 스트림을 받아 **실시간으로 줄 추가**. `speaking-line`은 발화 중 표시용 |
| AI 사고 흐름 | `aiThinkingEvents` 배열 + setTimeout | 동일한 이벤트 형태를 **WebSocket 실시간 이벤트**로 교체. 배열은 폴백/데모용으로 남겨도 됨 |
| 감정·확률 수치 (87%) | 하드코딩 | AI 모델 출력값으로 |
| 본인확인 체크리스트 | 체크박스 수동 | `verify_identity` 도구 결과로 자동 진행 |
| 도구 상태 (verify_identity) | 타이머로 상태 전환 | 실제 도구 호출 결과로 전환 |
| 카드 후보 / 고객 정보 | 하드코딩(김민서, 4821) | Mock 카드사 데이터에서 |
| 태블릿 device-frame 목업 | 검은 태블릿 테두리(그래픽) | **테두리 그래픽만 제거**하고 태블릿 화면에 **풀스크린**으로. 레이아웃은 태블릿 가로 기준 그대로 유지 |

## 3. 실시간 이벤트 계약 (WebSocket)
`aiui.html`의 `aiThinkingEvents` 구조를 그대로 이벤트 스키마로 승격. 서버→화면으로 흘려보낼 이벤트 예:
```
transcript   { speaker: "고객|AI|상담", text, partial: true|false }   // 현재 통화에 줄 추가
ai_step      { step: 0~4, reveal, type[], count?, toast }             // 오른쪽 흐름 노드 재생
tool_call    { tool: "verify_identity", state: "ready|running|done" } // 도구 상태
verify_done  { unlocked: true }                                       // 본인확인 완료 → 체크리스트 채움
caption      { text }                                                 // 하단 실시간 자막
```
- step 0 발화 감지 → 1 단서 추출 → 2 본인 확인 → 3 결과 수신 → 4 응답 결정 (기존 `workflowSteps`와 1:1 매핑).
- `tool_call: done` 이 오면 기존 `completePhone` 처럼 "등록 휴대폰 끝자리 확인" 체크가 대기→완료로.

## 4. 데모의 핵심 메시지 (놓치지 말 것)
**본인확인이 끝나면(verify_identity done) AI가 카드사 개인정보에 바로 접근 가능**해지는 편리함을 보여주는 화면이다. 그래서 본인확인 완료 = 개인정보/카드 정지 실행 잠금 해제로 연출. (실서비스 전환 시 실제 인증·암호화 필요하다는 주석만 코드에 남길 것 — 데모용임을 명시.)

## 5. 만들 것 (기술)
- **태블릿 앱** — 상담사는 **태블릿에서 가로 모드**로 사용한다. 3패널 가로 레이아웃을 태블릿 화면에 꽉 차게(풀스크린) 표시. `aiui.html`의 검은 태블릿 테두리(device-frame)는 **발표용 목업이므로 제거**하고, 안쪽 화면(`tablet-screen` 이하)만 실제 앱 화면으로 쓴다.
- 패키징: 기존 HTML/CSS를 살리려면 **Capacitor로 안드로이드 태블릿 앱**으로 감싸는 방식 권장(고객 앱과 동일 전략). 웹뷰로도 동일 화면 재사용 가능.
- **React + Vite + Tailwind** 로 `aiui.html`을 컴포넌트화(패널 3개 + 하단 자막). 색상·폰트·애니메이션은 기존 CSS 토큰 그대로 이식. 태블릿 가로 해상도 기준으로 그리드 고정.
- **WebSocket 클라이언트**로 3절 이벤트 수신 → 기존 재생 함수(typeText/countUp/setToolStatus 등)에 연결.
- 음성엔진·자체 LLM은 아직 없으니 **어댑터 뒤에 Mock**을 두고, 위 이벤트를 가짜로 쏘는 목업 서버로 데모가 돌아가게.
- (고객 앱 `user.html` · Mock 백엔드는 별도 인수인계 문서 참조 — 이 상담사 UI는 그 백엔드의 WebSocket에 붙는다.)

## 5.5 디자인 준수 기준 (반드시 지킬 가드레일)
**원칙: 내가 만든 `aiui.html` 디자인을 최대한 그대로 따른다.** 새 색/폰트/레이아웃을 창작하지 말 것. 아래 5개 축은 첨부 `ai_critique.json` 기준이며, 구현 후에도 이 상태를 유지해야 한다.

- **철학(philosophy)**: 오른쪽 AI 판단 패널은 "정적 표시"가 아니라 "실시간 사고 재생 흐름". 나머지 섹션 구조(현재 통화·상담 작업대)는 그대로 유지.
- **위계(hierarchy)**: 상단 5단계(발화 감지·단서 추출·본인 확인·결과 수신·응답 결정) 중 **현재 단계 하나만** 강조. 여러 개 동시 강조 금지.
- **실행(execution)**: 사고는 이벤트 배열+상태 머신으로 순차 재생 — 발화→감정 신호→대응 판단→추론 문장→`verify_identity` 실행 카드 순서 보존.
- **구체성(specificity)**: 불안 높음 **87% 카운트업**, 절차 안내/추가 질문은 **반투명 후보**로, 공감 대응이 선택되고, 등록 휴대폰 끝자리 확인 완료와 연동 — 이 디테일을 없애지 말 것.
- **절제(restraint)**: 밝은 배경 + 민트·라벤더 포인트 + 둥근 카드 + 얇은 테두리 유지. 모션은 짧은 ease-out으로 제한(과한 애니메이션 금지).

> 요약: 구조·색·모션은 손대지 말고, **데이터만 실제로 교체**. 리팩터링·컴포넌트화를 하더라도 렌더 결과가 `aiui.html`과 픽셀 수준에서 같아야 한다.
